Monorepo under `nudenet_project/` with three runtime layers orchestrated by `docker-compose.yml`:

- Backend (`backend/app/`) — FastAPI application layered as `api/` (route handlers: `auth`, `keys`, `moderate`, `analytics`) → `services/` (business logic: `ai_moderation`, `multi_model_moderation`, `video_moderation`, `hash_cache`) → `repositories/` (SQLAlchemy async repos: `key_repo`, `log_repo`, `user_repo`, `video_log_repo`) → `models/` (SQLAlchemy ORM: `User`, `ApiKey`, `ModerationLog`, `VideoModerationLog`). Shared infrastructure lives in `core/` (`config.py` via Pydantic Settings, `database.py` async engine, `security.py` JWT/OAuth guards, `celery_app.py`, `redis.py`, `rate_limit.py`, `advanced_rate_limit.py`).
- AI pipeline is centralized in `services/multi_model_moderation.py`: each detector (NSFW/NudeNet, violence/CLIP, weapons/YOLOv8, faces/MTCNN, text/PaddleOCR+profanity) is lazy-loaded through module-level `_xxx_detector` globals guarded by `get_xxx()` functions, then dispatched concurrently via `asyncio.gather` over a `ThreadPoolExecutor`; results are aggregated into a `ModerationResult` dataclass with an ensemble risk-score mapping (`low=0, medium=25, high=50, critical=100`).
- Persistence uses SQLAlchemy async with Alembic migrations under `backend/migrations/versions/`; default SQLite (`moderation.db`) switches to PostgreSQL via `DATABASE_URL` env.
- Async offloading: image moderation endpoints use `BackgroundTasks` for DB writes; batch URL moderation enqueues Celery tasks (`app.tasks.moderate_batch`) via `celery_app.send_task`; video moderation creates a `VideoModerationLog` row and runs frame extraction in-process via `run_video_moderation_job`.
- Frontend (`frontend/`) is a Vite + React + TypeScript SPA served by nginx; a Next.js backup exists under `frontend-nextjs-backup/`. Both talk to `/api/v1` on the backend.
- Deployment: Docker Compose spins up `postgres:15-alpine`, `redis:7-alpine`, two copies of the backend image (one running Uvicorn, one running `celery -A app.core.celery_app worker`), and the frontend container; healthchecks gate startup ordering.
